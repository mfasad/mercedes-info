Но даже с таким объёмом информации новичкам сложно сориентироваться: где скачать нужный файл, как правильно обновить систему без сбоев, или почему камеры отказываются подключаться.

Структура раздела техподдержки Trassir на dssl.ru: что где искать

Главная страница поддержки Trassir разделена на 5 ключевых блоков, каждый из которых решает конкретные задачи. Если вы впервые здесь, начните с верхнего меню:

  • 📥 Загрузки — прошивки, клиентские приложения (Trassir Client, Mobile Client), SDK для разработчиков и утилиты (например, Trassir License Manager).
  • 📄 Документация — мануалы по установке, API, схемы подключения оборудования (Dahua, Hikvision, Axis).
  • 🔧 FAQ и решения — база знаний с ответами на типичные ошибки (например, Error 1006 или проблемы с лицензией).
  • 📧 Обратиться в поддержку — форма для создания тикета (требуется регистрация на сайте DSSL).
  • 🔄 Архив версий — старые релизы ПО (полезно для отката или тестирования совместимости).

Важный нюанс: файлы в разделе Загрузки сортируются по дате, а не по актуальности. Например, прошивка Trassir 5.4.0.12345 может оказаться ниже, чем 5.3.0.9876, если её залили позже. Всегда проверяйте номер версии в имени файла перед скачиванием!

Как скачать и установить прошивку Trassir: пошаговая инструкция

Обновление серверной части Trassir — критически важная процедура, которую многие выполняют неправильно. Ошибки здесь ведут к потере настроек, отказу лицензий или даже "окирпичиванию" сервера. Следуйте этому алгоритму:

  1. Проверьте совместимость. Убедитесь, что новая версия поддерживает ваше "железо". Например, Trassir 5.x не работает на серверах с процессорами старше Intel Xeon E5-2600 v3.
  2. Скачайте правильный файл. Для сервера нужна прошивка с пометкой Server (например, Trassir_Server_5.4.0.12345.exe), для клиента — Client.
  3. Сделайте бэкап. Экспортируйте настройки через Администрирование → Экспорт/Импорт и сохраните лицензионный ключ (файл .lic).
  4. Отключите антивирус. Kaspersky и ESET часто блокируют установщики Trassir как "подозрительные".
  5. Запустите установку от имени администратора. Если обновляете удалённо, используйте PsExec:
psexec \\имя_сервера -u администратор -p пароль -h "путь_к_установщику.exe" /SILENT

После установки сервер перезагрузится автоматически. Не прерывайте процесс, даже если экран завис на 10+ минут — это нормально для больших баз данных (1000+ камер).

Распространённые ошибки Trassir и их решения (с кодами)

Система Trassir выдаёт ошибки в формате Error XXX, где XXX — числовой код. Ниже таблица с самыми частыми проблемами и способами их устранения:

Код ошибки Причина Решение
Error 1006 Нет связи с сервером Проверьте сетевой кабель, брандмауэр (порт 554 должен быть открыт) и службу TrassirServer в services.msc.
Error 2015 Недействительная лицензия Переустановите лицензию через Trassir License Manager или запросите новый ключ в поддержке DSSL.
Error 3004 Не хватает места на диске Очистите архив записей (Администрирование → Архив → Очистка) или добавьте новый диск в хранилище.
Error 4001 Несовместимая версия клиента и сервера Обновите клиентское ПО до версии сервера (или наоборот).
Error 5003 Ошибка базы данных Восстановите БД через Trassir Database Tool или обратитесь в поддержку с логами (C:\Trassir\Logs).

Если вашей ошибки нет в таблице, проверьте логи сервера по пути C:\Trassir\Logs\Server\. Ищите файлы с расширением .log и датой сегодняшнего дня. Для анализа логов используйте утилиту LogExpert или отправьте их в поддержку DSSL с описанием проблемы.

💡

При ошибке Error 2015 (лицензия) попробуйте сбросить время на сервере на 1–2 дня назад. Иногда это временно восстанавливает работу до получения нового ключа.

Настройка интеграции Trassir с камерами Dahua, Hikvision и Axis

Trassir поддерживает более 5000 моделей камер, но подключение каждого бренда имеет нюансы. Например, для Dahua требуется включить протокол ONVIF и указать правильные порты, а Axis часто конфликтует с шифрованием HTTPS.

Общая схема подключения:

  1. Добавьте камеру в Администрирование → Устройства → Добавить.
  2. Выберите производителя и модель из списка (если её нет, используйте ONVIF).
  3. Укажите IP-адрес, логин/пароль (по умолчанию для Hikvision — admin/12345).
  4. Настройте потоки:
    • 📹 Основной поток — высокое разрешение (например, 2560×1440 при 15 FPS).
    • 📱 Дополнительный поток — низкое разрешение (640×480 при 30 FPS) для мобильных клиентов.

Для Dahua дополнительно:

  • 🔌 Включите ONVIF в веб-интерфейсе камеры (Network → Advanced → Integration Protocol).
  • 🔒 Отключите Digested Authentication в настройках Trassir (если камера не подключается).

Для Axis:

  • 🛡️ Используйте протокол RTSP вместо ONVIF, если изображение тормозит.
  • 🔄 Обновите прошивку камеры до последней версии (через Axis Camera Management).
Как подключить камеру без поддержки ONVIF?

Если ваша камера не поддерживает ONVIF (например, старые модели Samsung Techwin), используйте прямой RTSP-поток. Формат URL: rtsp://IP_камеры:554/cam/realmonitor?channel=1&subtype=0. В Trassir выберите тип устройства "Универсальная IP-камера" и введите этот адрес в поле "Поток".

Оптимизация производительности Trassir для больших систем (100+ камер)

При масштабировании системы до 100+ камер Trassir начинает тормозить, если сервер не оптимизирован. Основные "узкие места":

  • 🖥️ ЦП — загрузка процессора выше 80% ведёт к лагам при просмотре архива.
  • 💾 Диски — HDD не справляются с записью 100+ потоков (нужны SSD или RAID-массивы).
  • 🌐 Сеть — 1 Гбит/с недостаточно для 4K-потоков от 50+ камер.

Рекомендации по оптимизации:

  1. Разделение ролей: используйте отдельные серверы для записи (Trassir Server) и аналитики (Trassir Analytics).
  2. Настройка архива:
    • 📅 Установите Глубину архива не более 30 дней для HD-камер (иначе диски переполнятся).
    • ⏱️ Включите Адаптивную запись (запись только при движении) для экономии места.
  • Оптимизация базы данных: 
    Trassir Database Tool --optimize --threshold 50

    (запускайте еженедельно для чистки фрагментированных данных).

    • 💡

    Для систем с 200+ камерами используйте кластерную архитектуру Trassir Enterprise с балансировкой нагрузки. Это позволяет распределить потоки между несколькими серверами и избежать простоя.

    Лицензирование Trassir: как активировать, продлить и восстановить ключ

    Лицензии Trassir привязываются к аппаратному ключу (HASP) или серийному номеру сервера. Если вы потеряли ключ или купили б/у сервер, восстановить лицензию можно через DSSL, но для этого понадобятся:

    • 📇 Серийный номер сервера (смотрите в Администрирование → О программе).
    • 📧 Данные покупателя (название организации, ИНН — если лицензия корпоративная).
    • 🔑 Код активации (если лицензия электронная, ищите в письме от DSSL).

    Процедура активации:

    1. Скачайте Trassir License Manager с сайта поддержки.
    2. Подключите HASP-ключ (если используется) к USB-порту сервера.
    3. Запустите License Manager и выберите Активировать лицензию.
    4. Введите серийный номер и код активации (или загрузите файл .lic).

    Если лицензия не активируется:

    • 🔌 Проверьте, горит ли индикатор на HASP-ключе (если нет — ключ неисправен).
    • ⏳ Убедитесь, что дата на сервере синхронизирована с интернетом (разница более 5 минут блокирует активацию).
    • 📡 Отключите VPN или прокси — они могут блокировать соединение с серверами DSSL.
    Что делать если HASP-ключ сломался?

    Если физический ключ повреждён, обратитесь в поддержку DSSL с фотографией ключа и чеком о покупке. Компания может перенести лицензию на новый ключ или выдать временный электронный ключ на 30 дней.

    Скрытые функции Trassir: что не пишут в документации

    Trassir имеет ряд недокументированных возможностей, которые упрощают администрирование. Вот некоторые из них:

    • 🔍 Поиск по лицу в архиве: в Trassir 5.x можно искать человека по фото, даже если модуль Face Recognition не активирован. Для этого загрузите фото в Аналитика → Поиск по изображению.
    • 📊 Экспорт статистики в Excel: отчёты по событиям (например, детекция движения) можно экспортировать в .xlsx через Администрирование → Отчёты → Экспорт.
    • 🔄 Удалённое управление через Telnet: подключитесь к серверу по Telnet (порт 23) и используйте команды: 
      system restart       # Перезагрузка сервера

      camera list         # Список подключённых камер
      

      license show         # Информация о лицензии

    • 🛡️ Скрытый режим админа: при установке Trassir создаётся скрытый пользователь superadmin с паролем dssl123 (работает только в локальной сети).

    Осторожно: недокументированные функции могут измениться в новых версиях ПО. Например, команда license show в Trassir 5.5+ требует прав суперпользователя.

    FAQ: ответы на частые вопросы по Trassir

    Как сбросить пароль администратора в Trassir?

    Если вы забыли пароль от учётной записи admin, сделайте следующее:

    1. Остановите службу TrassirServer через services.msc.
    2. Откройте файл C:\Trassir\Server\config.xml в блокноте.
    3. Найдите строку <Password>... и удалите её содержимое (между тегами).
    4. Сохраните файл и перезапустите службу.

    После этого пароль сбросится на пустой (или admin, в зависимости от версии).

    Можно ли использовать Trassir без лицензии?

    Да, но с ограничениями:

    • 🚫 Работает только 1 камера.
    • 🕒 Архив хранится не более 24 часов.
    • ❌ Отключены все модули аналитики (Face Recognition, ANPR и др.).

    Для тестирования можно запросить временную лицензию на 30 дней через форму на сайте DSSL.

    Как перенести Trassir на другой сервер?

    Процедура переноса:

    1. Экспортируйте настройки через Администрирование → Экспорт/Импорт.
    2. Скопируйте папку с архивом (C:\Trassir\Archive) на новый сервер.
    3. Установите ту же версию Trassir, что была на старом сервере.
    4. Импортируйте настройки и подключите лицензию (через License Manager).

    Если IP-адрес сервера изменился, обновите его в настройках камер и клиентских приложений.

    Почему Trassir не видит камеры по ONVIF?

    Проблемы с ONVIF обычно связаны с:

    • 🔌 Закрытым портом 3702 (проверьте брандмауэр).
    • 🔄 Несовпадением версий протокола (в Trassir и камере).
    • 📡 Неправильным логином/паролем (у некоторых камер Hikvision по умолчанию пустой пароль).

    Решение: используйте утилиту ONVIF Device Manager для тестирования подключения к камере.

    Как настроить уведомления о движении на email?

    Алгоритм настройки:

    1. Перейдите в Администрирование → События → Детектор движения.
    2. Создайте новое событие и выберите камеры для мониторинга.
    3. В разделе Действия добавьте Отправить email.
    4. Настройте SMTP-сервер (например, smtp.yandex.ru:465 с SSL).
    5. Укажите получателей и шаблон письма (можно вставить скриншот с камеры).

    Для теста отправьте письмо вручную через Тестовое событие.

    💡

    Перед обращением в поддержку DSSL всегда прикрепляйте к тикету логи сервера (C:\Trassir\Logs\Server\) и скриншот ошибки. Это ускорит решение проблемы в 3–5 раз.
