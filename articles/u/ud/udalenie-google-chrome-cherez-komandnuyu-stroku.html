Удаление Google Chrome через командную строку: полное руководство 2026

Удаление браузера Google Chrome через командную строку (CMD) — это не просто альтернатива стандартному способу через «Панель управления». Это мощный инструмент для полного удаления, когда браузер отказывается деинсталлироваться обычным путём, оставляет после себя «хвосты» в реестре или конфликтует с другими программами. Особенно актуально для системных администраторов, которые управляют парком компьютеров, или пользователей, столкнувшихся с критическими ошибками Chrome (например, ERR_CONNECTION_RESET или бесконечные обновления).

В этой статье вы найдёте не только базовые команды для удаления, но и уникальные скрипты для очистки кэша, профилей пользователей и записей в реестре, которые стандартные деинсталляторы часто пропускают. Мы также разберём типичные ошибки (например, Access Denied или File in Use) и способы их обхода без использования сторонних утилит. Если вы ищете способ избавиться от Chrome раз и навсегда — этот гайд для вас.

Почему командная строка эффективнее стандартного удаления?

Стандартная деинсталляция через «Параметры Windows» или appwiz.cpl часто оставляет после себя:

  • 📁 Папки профилей в %LocalAppData%\Google\Chrome с куками, историей и расширениями;
  • 🔑 Записи в реестре, которые могут конфликтовать с новой установкой;
  • 🗑️ Временные файлы в %Temp% и Prefetch;
  • 🔄 Задачи в планировщике (Task Scheduler), отвечающие за автоматическое обновление.

Командная строка позволяет:

  • ⚡ Удалить Chrome без запуска интерфейса (полезно для повреждённых систем);
  • 🔍 Очистить все связанные файлы одной командой;
  • 🛡️ Обойти блокировки антивирусов или групповой политики;
  • 📊 Автоматизировать процесс для массового удаления на нескольких ПК.

Кроме того, CMD даёт доступ к утилитам вроде wmic или msiexec, которые работают на более низком уровне, чем графические инструменты. Это особенно важно, если Chrome был установлен через MSI-пакет (например, в корпоративной среде) или его файлы защищены от изменений.

Подготовка системы перед удалением

Перед тем как приступить, выполните обязательные шаги:

  1. Закройте Google Chrome и все связанные процессы: 
    taskkill /f /im chrome.exe /t

    Флаг /t принудительно завершает все дочерние процессы (например, расширения или фоновые задачи).

  2. Отключите автоматическое обновление Chrome через планировщик задач: 
    schtasks /delete /tn "GoogleUpdateTaskMachineUA" /f

    schtasks /delete /tn "GoogleUpdateTaskMachineCore" /f

  3. Создайте точку восстановления системы (на случай ошибок в реестре): 
    wmic.exe /Namespace:\\root\default Path SystemRestore Call CreateRestorePoint "Удаление Chrome", 100, 7

Если вы используете Windows 10/11 с включённым Контролем учётных записей (UAC), запустите CMD от имени администратора. Для этого:

  1. Нажмите Win + X;
  2. Выберите «Терминал (администратор)» или «Командная строка (администратор)».

⚠️ Внимание: Если на ПК установлены другие программы от Google (например, Google Drive или Google Earth), их работа может быть нарушена после очистки общих компонентов. Проверьте зависимые приложения заранее.

Способы удаления Chrome через CMD

Существует три основных метода удаления Google Chrome через командную строку. Выбор зависит от того, как браузер был установлен и какие компоненты нужно очистить.

Метод Когда использовать Преимущества Недостатки
wmic Стандартная установка через .exe Удаляет основные файлы и записи в реестре Не очищает профили пользователей
msiexec Установка через .msi (корпоративная версия) Полное удаление с логами Требует знания ProductCode
Ручная очистка Повреждённая установка или глубокая очистка Удаляет все следы, включая кэш Требует прав администратора

Метод 1: Удаление через wmic

Самый простой способ для большинства пользователей. Команда автоматически находит Chrome в списке установленных программ и запускает его деинсталлятор:

wmic product where name="Google Chrome" call uninstall /nointeractive

Флаг /nointeractive подавляет диалоговые окна (полезно для скриптов). Если команда не сработала, попробуйте альтернативный вариант:

wmic product where "name like 'Google Chrome%'" call uninstall

Метод 2: Удаление через msiexec (для MSI-пакетов)

Корпоративные версии Chrome устанавливаются через .msi. Чтобы найти идентификатор продукта, выполните:

wmic product get name, identifyingnumber | find "Chrome"

Затем используйте команду:

msiexec /x {ProductCode} /qn /norestart

Замените {ProductCode} на реальный идентификатор (например, {ABC12345-6789-DEF0-1234-56789ABCDEF0}). Флаг /qn обеспечивает тихий режим без окон.

Метод 3: Ручная очистка (для опытных пользователей)

Если Chrome повреждён и не удаляется стандартными способами, примените полную очистку:

  1. Удалите папки: 
    rmdir /s /q "%ProgramFiles%\Google\Chrome"

    rmdir /s /q "%ProgramFiles(x86)%\Google\Chrome"
    

    rmdir /s /q "%LocalAppData%\Google\Chrome"
    

    rmdir /s /q "%UserProfile%\AppData\Roaming\Google\Chrome"

  2. Очистите реестр (с осторожностью!): 
    reg delete "HKLM\SOFTWARE\Google\Chrome" /f

    reg delete "HKLM\SOFTWARE\WOW6432Node\Google\Chrome" /f
    

    reg delete "HKCU\SOFTWARE\Google\Chrome" /f

  3. Удалите ярлыки: 
    del /f /q "%AppData%\Microsoft\Windows\Start Menu\Programs\Google Chrome.lnk"

    del /f /q "%Public%\Desktop\Google Chrome.lnk"

💡

Перед ручной очисткой реестра экспортируйте ветки через regedit (Файл → Экспорт). Это позволит восстановить данные при ошибке.

Ошибки при удалении и их решение

Даже при работе через CMD могут возникать ошибки. Рассмотрим самые распространённые и способы их исправления.

Ошибка 1: «Access Denied» (Отказано в доступе)

Причины:

  • 🔒 Недостаточно прав (запустите CMD от администратора);
  • 🛡️ Антивирус блокирует изменения (временно отключите защиту);
  • 🔄 Файлы используются другим процессом (проверьте через tasklist | find "chrome").

Решение:

takeown /f "C:\Program Files\Google\Chrome" /r /d y

icacls "C:\Program Files\Google\Chrome" /grant администраторы:F /t

Ошибка 2: «The system cannot find the file specified»

Это означает, что:

  • 📂 Папка Chrome уже удалена;
  • 🔍 Имя программы в команде указано неверно (проверьте через wmic product get name);
  • 🗑️ Установка была портативной (без регистрации в системе).

Ошибка 3: «Another version of this product is already installed»

Типична для msiexec. Решение:

  1. Найдите все версии Chrome: 
    wmic product where "name like 'Google%%'" get name, version
  2. Удалите их поочерёдно, начиная с самой старой.
⚠️ Внимание: Если после удаления Chrome перестаёт устанавливаться заново, проверьте наличие ключа HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Google\Chrome в реестре. Его присутствие может блокировать установку — удалите ветку вручную.

Дополнительная очистка: кэш, профили, задачи

Даже после деинсталляции Google Chrome может оставлять следы, которые:

  • 🕵️‍♂️ Отслеживают вашу активность (куки, история);
  • 🐢 Замедляют систему (кэш, временные файлы);
  • 🔄 Мешают новой установке (конфликтующие настройки).

Для полной очистки выполните:

1. Удаление профилей пользователей

Профили хранят личные данные, расширения и настройки. Чтобы удалить их для всех пользователей:

for /d %d in ("%LocalAppData%\Google\Chrome\User Data\*") do rd /s /q "%d"

2. Очистка кэша и временных файлов

Используйте встроенную утилиту cleanmgr:

cleanmgr /sagerun:1

Затем вручную удалите:

del /f /q "%LocalAppData%\Google\Chrome\Application\*.tmp"

del /f /q "%Temp%\Chrome*"

3. Удаление задач обновления

Google Update может продолжать работать даже после удаления браузера. Очистите его:

schtasks /delete /tn "GoogleUpdateTask*" /f

rmdir /s /q "%ProgramFiles%\Google\Update"


rmdir /s /q "%ProgramFiles(x86)%\Google\Update"

Как проверить, остались ли следы Chrome?

После очистки выполните поиск по ключевым словам:

dir /s /b "C:\" | find /i "chrome" > chrome_remnants.txt

Файл chrome_remnants.txt будет содержать пути ко всем оставшимся файлам и папкам с упоминанием Chrome. Удалите их вручную, если они не относятся к другим программам (например, Chrome Remote Desktop).

Автоматизация удаления (скрипты для администрирования)

Если вам нужно удалить Google Chrome на нескольких компьютерах (например, в офисе или школе), используйте пакетные файлы (.bat) или PowerShell-скрипты.

Скрипт 1: Простое удаление (BAT-файл)

Создайте файл uninstall_chrome.bat со следующим содержимым:

@echo off

:: Удаление через wmic


wmic product where name="Google Chrome" call uninstall /nointeractive



:: Очистка папок


rmdir /s /q "%ProgramFiles%\Google\Chrome" 2>nul


rmdir /s /q "%ProgramFiles(x86)%\Google\Chrome" 2>nul


rmdir /s /q "%LocalAppData%\Google\Chrome" 2>nul



:: Удаление ярлыков


del /f /q "%Public%\Desktop\Google Chrome.lnk" 2>nul


del /f /q "%AppData%\Microsoft\Windows\Start Menu\Programs\Google Chrome.lnk" 2>nul



echo Chrome удалён. Перезагрузите компьютер.


pause

Скрипт 2: Полная очистка (PowerShell)

Для более глубокой очистки используйте PowerShell:

# Удаление через MSI (если есть)

$chromeMSI = Get-WmiObject -Class Win32_Product | Where-Object { $_.Name -like "*Chrome*" }


if ($chromeMSI) { $chromeMSI.Uninstall() }



# Очистка папок


$paths = @(


"$env:ProgramFiles\Google\Chrome",


"$env:ProgramFiles(x86)\Google\Chrome",


"$env:LocalAppData\Google\Chrome",


"$env:USERPROFILE\AppData\Roaming\Google\Chrome"


)


foreach ($path in $paths) {


if (Test-Path $path) { Remove-Item -Recurse -Force $path }


}



# Удаление задач


Get-ScheduledTask | Where-Object { $_.TaskName -like "*Google*" } | Unregister-ScheduledTask -Confirm:$false



Write-Host "Очистка завершена. Перезагрузите ПК." -ForegroundColor Green

Чтобы запустить скрипт, сохраните его как uninstall_chrome.ps1 и выполните в PowerShell от администратора:

Set-ExecutionPolicy Bypass -Scope Process -Force

.\uninstall_chrome.ps1

💡

Для массового удаления в домене используйте Group Policy или Microsoft Endpoint Configuration Manager. Скрипты можно развернуть через psexec или Invoke-Command в PowerShell.

FAQ: Частые вопросы по удалению Chrome через CMD

Можно ли удалить Chrome без прав администратора?

Частично. Вы сможете удалить профиль пользователя и некоторые файлы в %LocalAppData%, но для полной деинсталляции (особенно очистки Program Files и реестра) требуются права администратора. Альтернатива — использовать портативную версию Chrome, которая не требует установки.

Как удалить Chrome, если он не отображается в списке программ?

Это значит, что браузер был установлен некорректно или через портативный архив. В таком случае:

  1. Проверьте папки %ProgramFiles% и %LocalAppData% на наличие папок Google\Chrome;
  2. Удалите их вручную (см. Метод 3 в статье);
  3. Проверьте Планировщик задач на наличие задач GoogleUpdate.
Почему после удаления Chrome снова появляется при обновлении Windows?

Это происходит из-за интеграции Chrome с компонентами Windows (например, WebView2) или политик организации. Чтобы предотвратить автоматическую установку:

  1. Отключите Google Update (см. раздел Дополнительная очистка);
  2. Используйте Group Policy (для Windows Pro/Enterprise):
    3. gpedit.msc → Конфигурация компьютера → Административные шаблоны → Компоненты Windows → Microsoft Edge → Отключить перенаправление сайтов из Internet Explorer в Microsoft Edge
  3. Установите Chrome Enterprise Bundle с отключёнными автоматическими обновлениями.
Как удалить Chrome вместе с всеми данными (историей, паролями)?

Для полного стирания личных данных:

  1. Удалите папку профиля: 
    rmdir /s /q "%LocalAppData%\Google\Chrome\User Data\Default"
  2. Очистите кэш DNS (если использовались прокси): 
    ipconfig /flushdns
  3. Для безвозвратного удаления (без возможности восстановления) используйте утилиту cipher: 
    cipher /w:"%LocalAppData%\Google"

⚠️ Внимание: Это удалит все сохранённые пароли, закладки и расширения без возможности восстановления. Экспортируйте данные заранее через chrome://settings/exportData.

Можно ли удалить Chrome, если он используется другой программой?

Если Chrome заблокирован (например, Steam, Discord или Electron-приложения используют его компоненты), выполните:

  1. Закройте все программы, которые могут использовать chrome.exe;
  2. Используйте Process Explorer (из Sysinternals) для поиска процессов, блокирующих файлы;
  3. Принудительно удалите файлы при следующей загрузке с помощью pendmoves:
    4. reg add "HKLM\SYSTEM\CurrentControlSet\Control\Session Manager" /v PendingFileRenameOperations /t REG_MULTI_SZ /d "\??\C:\Program Files\Google\Chrome\chrome.exe\nul" /f
