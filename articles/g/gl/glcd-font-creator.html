Работа с графическими дисплеями (GLCD, TFT, OLED) часто требует нестандартных шрифтов — будь то кириллица, специальные символы или оптимизированные под маленький экран значки. GLCD Font Creator (или GLCDFontCreator) становится незаменимым инструментом для инженеров, радиолюбителей и разработчиков встраиваемых систем, позволяя создавать шрифты в бинарном или графическом формате, совместимом с большинством микроконтроллеров. Однако без понимания нюансов работы с программой даже простая задача — например, добавление русского алфавита — может превратиться в часовую головоломку.

В этой статье мы разберём все этапы работы — от установки и базовой настройки до экспорта шрифтов для Arduino, STM32, ESP32 и других платформ. Особое внимание уделим оптимизации памяти (критичной для устройств с ограниченными ресурсами) и обходу типичных ошибок, из-за которых шрифт отображается иероглифами или вообще не загружается. Если вы когда-нибудь сталкивались с проблемами вроде "avr-gcc: error: undefined reference to 'font'", здесь вы найдёте решения.

Что такое GLCD Font Creator и зачем он нужен

GLCD Font Creator — это бесплатная утилита с открытым исходным кодом, предназначенная для генерации шрифтов в форматах, совместимых с графическими дисплеями. Программа позволяет:

  • 🎨 Рисовать символы вручную (пиксель за пикселем) или импортировать их из изображений (.bmp, .png).
  • 📏 Настраивать размеры шрифта (ширина × высота в пикселях, межсимвольный интервал).
  • 🖥️ Экспортировать в форматы для C/C++ (массивы байт), Python, или бинарные файлы.
  • 🔄 Поддерживать кодировки, включая UTF-8, Windows-1251, KOI8-R (актуально для кириллицы).

Главное преимущество программы — гибкость. Например, вы можете создать шрифт высотой 8 пикселей для Nokia 5110 или 32 пикселя для ILI9341, при этом оптимизировав его под конкретный контроллер. Без такого инструмента пришлось бы вручную прописывать каждый символ в виде массива байт, что занимает часы и чревато ошибками.

⚠️ Внимание: GLCD Font Creator не поддерживает антиалиасинг (сглаживание краёв). Если вам нужны плавные шрифты, рассмотрите альтернативы вроде LVGL Font Tool или Bitmap Font Generator.

Программа особенно востребована в проектах на:

  • 🤖 Arduino (библиотеки U8g2, Adafruit_GFX).
  • 🔧 STM32 (с использованием STM32CubeIDE или Keil).
  • 📡 ESP8266/ESP32 (для дисплеев по SPI или I2C).
  • 🖥️ Промышленных ПЛК (где требуются кастомные символы для интерфейса оператора).

Установка и первый запуск

Программа GLCD Font Creator не требует установки — достаточно скачать архив с официального репозитория на GitHub (автор — Olikraus, тот же, кто разработал библиотеку U8g2). Актуальная версия на момент написания статьи — 1.2.1.

После распаковки архива:

  1. Запустите файл GLCDFontCreator.jar (требуется установленная Java 8+).
  2. Если Java отсутствует, скачайте её с Adoptium (рекомендуется OpenJDK 11).
  3. При первом запуске программа создаст конфигурационный файл config.properties в той же папке.

Интерфейс программы интуитивно понятен, но имеет особенности:

  • 🖼️ Панель символов (слева) — здесь отображаются все доступные glyphs (по умолчанию — латиница и цифры).
  • 📝 Редактор символа (справа) — сетка для рисования пикселей.
  • ⚙️ Панель настроек (внизу) — параметры шрифта, кодировка, экспорт.
⚠️ Внимание: Если при запуске появляется ошибка "Unable to access jarfile", проверьте, что путь к файлу не содержит кириллических символов или пробелов. Перенесите папку в корень диска (например, C:\GLCDFontCreator\).

Для проверки работоспособности:

  1. Выберите символ A в панели слева.
  2. В редакторе (справа) нарисуйте произвольный узор.
  3. Нажмите File → Export → C Source File и сохраните файл как test_font.c.
💡

Если программа не запускается на Windows 10/11, попробуйте запустить её от имени администратора или установите переменную окружения JAVA_HOME, указав путь к папке с Java.

Создание и редактирование шрифта

Прежде чем приступать к рисованию, определитесь с параметрами шрифта:

  • 📏 Размер: стандартные значения — 5×7, 8×8, 16×16. Для OLED-дисплеев (например, SSD1306) часто используют 6×8.
  • 🔤 Кодировка: для кириллицы выберите Windows-1251 или UTF-8 (последняя поддерживает больше символов, но требует больше памяти).
  • 🎯 Межсимвольный интервал (Spacing): по умолчанию 1 пиксель. Увеличьте до 2–3 для лучшей читаемости.

Процесс создания шрифта:

  1. В меню File → New укажите имя проекта (например, MyFont_8x16).
  2. В панели Font Properties установите: 
    Width: 8

    Height: 16
    

    Spacing: 2
    

    Encoding: Windows-1251 (для русского)

  3. Добавьте нужные символы:
    • 🔤 Для латиницы: диапазон 32–126 (ASCII).
    • 🇷🇺 Для кириллицы: диапазон 192–255 (Windows-1251) или 1024–1103 (UTF-8).
  • Нарисуйте символы вручную или импортируйте из .bmp (меню File → Import → Bitmap).

    • При ручном рисовании используйте горячие клавиши:

    • ЛКМ — установить пиксель.
    • ПКМ — стереть пиксель.
    • Shift + ЛКМ — заливка области.
    • Пробел — переключить цвет (черный/белый).

    Экспорт шрифта для микроконтроллеров

    После создания шрифта его нужно экспортировать в формат, понятный вашему микроконтроллеру. Программа поддерживает несколько вариантов:

    Формат экспорта Описание Пример использования
    C Source File Массив байт в виде .c/.h файлов. Подходит для Arduino, STM32. 
    const uint8_t myFont[] PROGMEM = {

    0x00, 0x00, 0x00, ... };
    Binary File Бинарный файл (.bin), который можно прошить во flash-память. Промышленные ПЛК, где шрифт хранится в отдельном чипе.
    Python List Список байт для скриптов на Python (например, для Raspberry Pi). 
    font = [

    [0x00, 0x00, 0x00],
    

    ... ]
    U8g2 Format Специальный формат для библиотеки U8g2 (оптимизирован под Arduino). Дисплеи SSD1306, SH1106.

    Для большинства проектов подходит C Source File. Алгоритм экспорта:

    1. Выберите File → Export → C Source File.
    2. Укажите имя файла (например, my_font.h).
    3. В поле Variable Name введите имя массива (например, customFont).
    4. Отметьте галочку PROGMEM (если шрифт будет храниться во flash-памяти Arduino).
    5. Нажмите Export.

    Пример сгенерированного кода:

    #include 


    const uint8_t customFont[] PROGMEM = {
    

    // Символ 'A' (код 65)
    

    0x00, 0x1C, 0x22, 0x22, 0x22, 0x3E, 0x22, 0x22, 0x22, 0x00,
    

    // Символ 'Б' (код 194 в Windows-1251)
    

    0x00, 0x1C, 0x22, 0x22, 0x3E, 0x22, 0x22, 0x22, 0x3E, 0x00,
    

    ...
    

    };

    ⚠️ Внимание: Если вы экспортируете шрифт для STM32 с использованием STM32CubeIDE, удалите директиву #include — она предназначена только для AVR (Arduino). Вместо неё используйте #include "stm32fxxx_hal.h" и разместите массив в секции FLASH:

    const uint8_t customFont[] __attribute__((section(".rodata"))) = { ... };

    Интеграция шрифта в проект

    После экспорта шрифт нужно подключить к вашему проекту. Рассмотрим два популярных сценария: Arduino и STM32.

    Для Arduino (с библиотекой U8g2)

    Если вы использовали формат U8g2:

    1. Скопируйте сгенерированный файл my_font.h в папку проекта.
    2. В основном скетче подключите библиотеку и шрифт: 
      #include 

      #include "my_font.h"
      


      U8G2_SSD1306_128X64_NONAME_F_HW_I2C u8g2(U8G2_R0, /* reset=*/ U8X8_PIN_NONE);
      


      void setup() {
      

      u8g2.begin();
      

      u8g2.setFont(customFont); // Используем наш шрифт
      

      u8g2.drawStr(0, 20, "Привет, мир!");
      

      u8g2.sendBuffer();
      

      }

    Для STM32 (с использованием HAL)

    Для STM32 процесс сложнее из-за ручной работы с дисплеем:

    1. Добавьте файл my_font.h в проект STM32CubeIDE.
    2. В функции отображения текста используйте массив шрифта: 
      void LCD_DrawChar(uint16_t x, uint16_t y, char ch) {

      uint8_t *char_data = (uint8_t *)&customFont[ch * FONT_HEIGHT];
      

      for (uint8_t i = 0; i < FONT_HEIGHT; i++) {
      

      uint8_t line = char_data[i];
      

      for (uint8_t j = 0; j < FONT_WIDTH; j++) {
      

      if (line & (1 << (7 - j))) {
      

      LCD_DrawPixel(x + j, y + i, WHITE);
      

      }
      

      }
      

      }
      

      }

    Для ESP32 с библиотекой Adafruit_GFX процесс аналогичен Arduino, но вместо PROGMEM используйте const:

    const uint8_t customFont[] = { ... };
    💡

    Всегда проверяйте, что размер массива шрифта не превышает доступную память микроконтроллера. Для Arduino Uno (2 КБ SRAM) шрифт размером 16×16 с 256 символами займёт ~8 КБ flash-памяти!

    Оптимизация памяти и производительности

    Шрифты могут занимать значительную часть памяти микроконтроллера. Например, шрифт 16×16 с 256 символами требует:

    (16 пикселей высота × 2 байта на строку × 256 символов) = 8192 байт (~8 КБ).

    Чтобы сократить расход памяти:

    • 🧹 Удалите ненужные символы. Например, если вам нужна только кириллица и цифры, экспортируйте диапазон 48–57 (цифры) и 192–255 (Windows-1251).
    • 🔍 Используйте сжатие. В настройках экспорта выберите RLE Compression (Run-Length Encoding). Это сокращает размер на 30–50%, но увеличивает время рендеринга.
    • 📉 Уменьшите высоту шрифта. Например, вместо 16×16 используйте 8×16 (ширина в 2 раза меньше).
    • 🔄 Храните шрифт во flash (не в SRAM). Для этого используйте PROGMEM (Arduino) или секцию .rodata (STM32).

    Пример оптимизированного экспорта для Arduino:

    // В файле my_font.h

    #include 
    


    const uint8_t customFont[] PROGMEM = {
    

    // Только символы '0'–'9' и 'А'–'Я' (кириллица)
    

    // Размер: 10 (цифры) + 32 (кириллица) = 42 символа × 16 байт = 672 байт
    

    ...
    

    };

    Для STM32 можно пойти дальше и хранить шрифт в внешней flash-памяти (например, W25Q128), загружая его по мере необходимости. Это актуально для проектов с дисплеями высокого разрешения (например, ILI9341 320×240).

    Как проверить занятую память в Arduino IDE?

    Откройте Инструменты → Анализ использования памяти (Tools → Get Board Info в новых версиях). В выводе найдите строки:

    Global variables use 1234 bytes (60%) of dynamic memory

    Program storage space used: 23456 bytes (71%)

    Если Program storage близко к 100%, уменьшите размер шрифта или удалите ненужные символы.

    Типичные ошибки и их решения

    Даже опытные разработчики сталкиваются с проблемами при работе с GLCD Font Creator. Вот самые распространённые:

    Ошибка Причина Решение
    Шрифт отображается иероглифами Неверная кодировка (например, выбрана UTF-8, а нужен Windows-1251). Проверьте настройки кодировки в Font Properties → Encoding.
    undefined reference to 'font' Файл со шрифтом не подключён к проекту или неправильное имя массива. Убедитесь, что:
    • Файл my_font.h лежит в папке проекта.
    • Имя массива в коде совпадает с экспортированным.
    • В Arduino IDE файл подключён через #include "my_font.h".
    Символы слипаются или обрезаются Неправильный Spacing или Width. Увеличьте Spacing до 2–3 или проверьте ширину символов в настройках.
    Программа не запускается на macOS/Linux Проблемы с Java или правами доступа. Запустите из терминала: 
    chmod +x GLCDFontCreator.jar

    java -jar GLCDFontCreator.jar

    Ещё одна частая проблема — несовместимость с библиотеками. Например, если вы используете U8g2, но шрифт экспортировали в обычном C-формате, символы могут отображаться со сдвигом. Решение:

    1. Экспортируйте шрифт в формате U8g2 (опция в меню экспорта).
    2. Или вручную отредактируйте сгенерированный файл, добавив заголовок: 
      #define FONT_WIDTH 8

      #define FONT_HEIGHT 16
      

      const uint8_t customFont[256][FONT_HEIGHT] PROGMEM = { ... };

    Если шрифт отображается "лесенкой" (неровные края), проверьте:

    • 🔍 Соответствие высоты шрифта разрешению дисплея (например, для SSD1306 128×64 лучше использовать шрифт высотой 8 или 16).
    • 🖼️ Правильность экспортированных данных (иногда биты в байтах перевёрнуты; попробуйте инвертировать их вручную).

    Альтернативы GLCD Font Creator

    Если GLCD Font Creator не подходит для вашей задачи, рассмотрите альтернативы:

    Программа Плюсы Минусы Лучше подходит для
    LVGL Font Tool
    • Поддержка сглаживания (антиалиасинг).
    • Экспорт в форматы для LVGL (популярная библиотека для GUI).
    		 Сложнее в освоении, требует настройки. Проекты с графическим интерфейсом на STM32, Raspberry Pi.
    Bitmap Font Generator (BFG)
    • Поддержка TrueType шрифтов (конвертация в растр).
    • Больше опций для кернинга и интерлиньяжа.
    		 Платная версия имеет больше функций. Профессиональные проекты с высокими требованиями к типографике.
    MicroFont
    • Простой веб-интерфейс (работает в браузере).
    • Поддержка кириллицы "из коробки".
    		 Ограниченные настройки экспорта. Быстрое создание шрифтов для Arduino.
    FontForge + скрипты
    • Максимальная гибкость (можно редактировать векторы).
    • Поддержка всех кодировок.
    		 Сложный для новичков, требует знания Python для автоматизации. Кастомные шрифты для промышленных дисплеев.

    Если вам нужна кириллица, обратите внимание на MicroFont или LVGL Font Tool — они лучше справляются с кодировками, чем GLCD Font Creator. Для ESP32 с дисплеями TFT (например, ILI9341) часто используют LVGL, поэтому LVGL Font Tool станет оптимальным выбором.

    Пример конвертации шрифта из .ttf в растр с помощью FontForge:

    1. Откройте .ttf-файл в FontForge.
    2. Выделите нужные символы (например, кириллицу).
    3. Экспортируйте в .bmp через File → Export → Bitmap.
    4. Импортируйте .bmp в GLCD Font Creator для финальной доводки.

    FAQ: ответы на частые вопросы

    Как добавить кириллицу в GLCD Font Creator?

    Чтобы добавить русские буквы:

    1. В настройках шрифта (Font Properties) выберите кодировку Windows-1251.
    2. Вручную добавьте символы с кодами 192–255 (соответствуют кириллице в Windows-1251).
    3. Нарисуйте каждый символ или импортируйте из .bmp.
    4. При экспорте убедитесь, что в коде указан правильный диапазон: 
      // В файле my_font.h

      #define FONT_FIRST_CHAR 32   // Пробел
      

      #define FONT_LAST_CHAR 255   // Последний символ (включает кириллицу)

    Если буквы отображаются неверно, проверьте кодировку терминала/дисплея — она должна совпадать с Windows-1251.

    Можно ли использовать GLCD Font Creator для OLED-дисплеев (SSD1306, SH1106)?

    Да, программа подходит для любых графических дисплеев, включая OLED. Главное — правильно настроить размер шрифта:

    • Для SSD1306 (128×64) оптимальный размер — 6×8 или 8×16.
    • Для SH1106 (128×64) — аналогично.
    • Экспортируйте шрифт в формате U8g2, если используете библиотеку U8g2.

    Пример кода для SSD1306:

    #include 

    #include "my_font.h"
    


    U8G2_SSD1306_128X64_NONAME_F_HW_I2C u8g2(U8G2_R0);
    


    void setup() {
    

    u8g2.begin();
    

    u8g2.setFont(my_font_8x16); // Подключаем шрифт
    

    u8g2.drawStr(0, 20, "Тест кириллицы");
    

    u8g2.sendBuffer();
    

    }

    Как уменьшить размер шрифта для Arduino Uno (ограничение 32 КБ flash)?

    Для Arduino Uno (ATmega328P) с 32 КБ flash-памяти:

    • 🧹 Удалите ненужные символы. Оставьте только цифры (48–57) и нужные буквы.
    • 📉 Уменьшите высоту. Вместо 16×16 используйте 8×8.
    • 🔄 Используйте сжатие RLE (в настройках экспорта).
    • 🗑️ Храните
