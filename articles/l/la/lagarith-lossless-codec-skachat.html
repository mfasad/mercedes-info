Что такое Lagarith Lossless Codec и зачем он нужен

Lagarith — это кодек сжатия видео без потерь, разработанный Беном Рудьяком-Гулдом в 2006 году. Его ключевое преимущество — поддержка RGB, YUY2 и YV12 цветовых пространств с возможностью выбора метода сжатия: от сверхбыстрого Null (фактически без сжатия) до High Compression с использованием алгоритма arithmetic coding. Кодек идеально подходит для:

  • 🎮 Захвата игрового геймплея в оригинальном качестве (например, для Speedrun-сообщества, где важна каждый пиксель).
  • 🎬 Промежуточного рендеринга в Adobe Premiere или Vegas Pro перед финальным сжатием в H.265.
  • 🔍 Архивирования видео с веб-камер или научных экспериментов, где потеря данных недопустима.
  • 🖥️ Виртуальной съёмки (например, через NVIDIA ShadowPlay или AMD ReLive).

В отличие от HuffYUV, который часто перегружает процессор, Lagarith распределяет нагрузку более равномерно. Например, при записи 1440p144 в CS:GO кодек потребляет на 15–20% меньше ресурсов CPU, чем MagicYUV, при сопоставимом размере файла. Это критично для стримеров, которые одновременно кодируют поток для Twitch и записывают локальный бэкап.

Однако у Lagarith есть и ограничения:

  • ⚠️ Нет аппаратного ускорения (работает только на CPU).
  • ⚠️ Последняя официальная версия (1.3.27) датируется 2013 годом, но остаётся стабильной.
  • ⚠️ Не поддерживает 10-bit цвет (в отличие от UT Video Codec).

Настройка Lagarith в OBS Studio для записи без потерь

Если вы стримите или записываете геймплей, настройка Lagarith в OBS Studio требует особого подхода. По умолчанию OBS использует x264 или NVENC, которые сжимают видео с потерями. Чтобы переключиться на Lagarith:

Шаг 1. Выбор кодека в настройках записи

  1. Откройте Файл → Настройки → Выход.
  2. В разделе Запись выберите: 
    Тип выхода: Стандартный

    Формат записи: AVI
    

    Кодек видео: Lagarith Lossless Codec

  3. Установите Качество (CQ) на Lossless (если поле активно).

Шаг 2. Оптимизация производительности

Чтобы избежать лагов при записи 1440p144 или 4K30:

  • В Настройки → Видео установите Выходное разрешение равным Базовому (без масштабирования).
  • В Настройки → Дополнительно выберите Процесс: Высокий приоритет.
  • Отключите VSync в играх, чтобы избежать конфликтов с записью.

Шаг 3. Проверка результата

После записи откройте файл в MediaInfo и убедитесь, что:

  • Формат видео: Lagarith.
  • Цветовое пространство: YUV 4:2:0 (если выбирали YV12).
  • Битрейт: ~500–800 Mbps для 1080p60.
⚠️ Внимание: При записи в Lagarith OBS может "просаживать" FPS в играх на 5–10% даже на мощных ПК. Если это критично — снизьте разрешение записи до 1280×720 или используйте NVENC с высоким битрейтом (50–80 Mbps).

Эта ошибка возникает, если Lagarith не зарегистрирован в системе как DirectShow-фильтр. Решение:

Использование Lagarith с VirtualDub и FFmpeg

VirtualDub остаётся одним из самых популярных инструментов для работы с Lagarith благодаря поддержке фильтров и пакетной обработки. Вот как настроить кодек для захвата и конвертации:

Захват видео через VirtualDub:

  1. Откройте VirtualDub и выберите File → Capture AVI.
  2. В Video → Compression выберите Lagarith Lossless Codec.
  3. Нажмите Configure и установите: 
    Mode: High Compression

    Output colorspace: YV12
    

    Use multithreading: ✅

  4. Для захвата с экрана используйте File → Capture AVI → Set capture file и настройте источник (например, Desktop Duplication API).

Конвертация через FFmpeg:

Lagarith поддерживается в FFmpeg через библиотеку libavcodec. Примеры команд:

  • Закодировать видео в Lagarith: 
    ffmpeg -i input.mp4 -c:v lagarith -pix_fmt yuv420p output.avi
  • Декодировать Lagarith в H.264: 
    ffmpeg -i input.avi -c:v libx264 -crf 18 -preset slow output.mp4
  • Извлечь кадры в PNG (для редактирования): 
    ffmpeg -i input.avi -vf "fps=30" frame_%04d.png

Проблемы и решения:

  • 🔴 "Unknown encoder 'lagarith'" — обновите FFmpeg до версии 4.4+.
  • 🔴 Видео в VirtualDub идёт рывками — отключите Drop frames if behind в настройках захвата.
  • 🔴 Цвета искажены — проверьте, что в настройках Lagarith и в FFmpeg используется одинаковое цветовое пространство (yuv420p или rgb24).

Оптимизация Lagarith для 4K и высоких FPS

Запись видео в разрешении 3840×2160 (4K) или с частотой 120+ FPS предъявляет высокие требования к кодеку. Lagarith справляется с этой задачей, но требует правильных настроек:

  • Null — минимальная нагрузка на CPU, но файлы огромные (~1.2 TB/час для 4K60).
  • Normal — баланс (рекомендуется для большинства задач).
  • High Compression — файлы на 20% меньше, но FPS может падать на 10–15%.

2. Настройка системы:

  • Отключите Game DVR в Windows (Настройки → Игры → Запись игровых клипов).
  • Установите приоритет процесса obs64.exe или VirtualDub.exe на Высокий через Диспетчер задач.
  • Используйте RAM-диск (например, ImDisk) для временного хранения файлов, если записываете на HDD.

Частые ошибки и их решения

Даже после правильной установки Lagarith может работать нестабильно. Мы собрали топ-5 ошибок и способы их исправления:

1. "No compatible streams found" в Media Player Classic

Причина: Отсутствует фильтр HAALI Media Splitter или конфликт с LAV Filters.

Решение: Установите K-Lite Codec Pack Standard (выбрав только LAV Filters и HAALI) или вручную зарегистрируйте фильтры:

regsvr32 haali.dll

regsvr32 lavfilters.dll

2. Видео воспроизводится с артефактами (зелёные полосы, рваные кадры)

Причина: Несовпадение цветового пространства между кодеком и плеером.

Решение: В настройках Lagarith установите Output colorspace: YV12 и используйте плеер с поддержкой DirectShow (например, MPC-BE).

3. OBS вылетает при старте записи с Lagarith

Причина: Конфликт с NVIDIA ShadowPlay или AMD ReLive.

Решение: Отключите фоновую запись в настройках графических драйверов или добавьте OBS в исключения.

4. Файлы Lagarith не открываются в Premiere Pro

Причина: Adobe не поддерживает Lagarith "из коробки".

Решение: Установите плагин AVI Importer или конвертируйте видео в DNxHD через FFmpeg:

ffmpeg -i input.avi -c:v dnxhd -profile:v dnxhr_hq -pix_fmt yuv422p output.mov

5. Запись прерывается через 2–4 GB

Причина: Ограничение формата AVI на 32-битные системы.

Решение: Используйте VirtualDub с плагином AVI File Handler или записывайте в MKV через OBS (требуется дополнительный мультиплексор).

💡

Если Lagarith работает нестабильно, первым делом проверьте конфликты с другими кодеками через утилиту Codec Tweak Tool. Она покажет все зарегистрированные фильтры и позволит отключить ненужные.

FAQ: Ответы на частые вопросы

🔹 Можно ли использовать Lagarith на macOS или Linux?

Официально Lagarith доступен только для Windows. Однако есть обходные пути:

  • На macOS можно использовать Wine для запуска VirtualDub с Lagarith, но производительность будет низкой.
  • На Linux альтернатива — FFmpeg с кодеком ffv1 или huffyuv: 
    ffmpeg -i input.mp4 -c:v ffv1 output.avi

Для полноценной работы рекомендуем виртуальную машину с Windows.

🔹 Как уменьшить размер файлов Lagarith без потерь?

Lagarith уже использует сжатие без потерь, но размер файла можно сократить на 10–20%:

  • В настройках кодека выберите High Compression (медленнее, но эффективнее).
  • Используйте YV12 вместо RGB (размер уменьшится на ~30%).
  • Если видео статичное (например, запись экрана с документами), снизьте FPS до 30.

Для архивного хранения можно дополнительно запаковать файл в 7-Zip с алгоритмом LZMA2 — это даст ещё ~5% сжатия.

🔹 Поддерживает ли Lagarith аппаратное ускорение (NVENC/AMF)?

Нет, Lagarith работает только на CPU. Если вам нужно аппаратное ускорение для записи без потерь, рассмотрите:

  • NVIDIA NVENC в режиме Lossless (доступно на RTX 20/30/40 серии).
  • AMD AMF с настройкой Quality: Lossless (требует драйвер Adrenalin 22.5.1+).
  • Intel QSV (только для 11-го поколения+).

Однако эти решения дают псевдо-lossless (технически есть минимальные потери), тогда как Lagarith гарантирует 100% сохранность данных.
